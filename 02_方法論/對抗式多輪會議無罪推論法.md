# 作業流程表 — 刑事案件無罪推論方法論（金恆通案實證版）

**方法論名稱：** 對抗式多輪會議無罪推論法（Adversarial Multi-Round Acquittal Analysis）
**首次實證：** 賴宗杰／金恆通金流涉詐洗錢賭博案（111偵10321、111偵15336）
**核心理念：** 以五角色對抗辯證 × 多輪迭代 × 證據驅動的結論修正，把「模糊特徵」逐一
拆解為可被定義消除或證據壓制的點，建立可在法庭站得住的無罪路徑。

---

## 階段總覽（七階段）

| 階段 | 名稱 | 目的 | 產出 |
|------|------|------|------|
| 0 | 資料匯入與索引 | 把卷證變成可檢索的結構 | 文件索引表 |
| 1 | 團隊建置 | 建立五角色對抗框架＋當事人人格 | Agent 團隊 |
| 2 | 特徵點盤點 | 窮舉偵查機關認定的所有「關聯特徵」 | 特徵點分析表 |
| 3 | 多輪對抗會議 | 用對抗辯證迭代修正每個特徵的結論 | 各輪會議紀錄 |
| 4 | 槓桿點識別與破防 | 找出決定全案的關鍵點（D 點）並攻破 | 槓桿點深化分析 |
| 5 | 事實校準 | 以當事人提供的真實事實推翻誤判 | 校準後重評 |
| 6 | 無罪路徑收斂 | 收斂成可執行的法庭策略＋誠實機率 | 無罪路徑總圖 |

## 階段 0：資料匯入與索引

| 步驟 | 動作 | 要點 |
|------|------|------|
| 0-1 | 從來源（Google Drive 等）列出全部文件 | 用 metadata 抓檔名、ID、類型 |
| 0-2 | 依「人物 × 文件類型 × 時序」建索引表 | 筆錄按受詢問人分組、按日期排序 |
| 0-3 | 標記核心調查標的與案號 | 涉案系統、關鍵公司、案號 |
| 0-4 | 建立研究方向初稿 | 為後續特徵盤點定錨 |

**產出：** `文件索引表`（見 `00_案件總覽/卷證文件索引.md`）

## 階段 1：團隊建置（五角色對抗框架）

| 角色 | 代號 | 立場 | 職能 |
|------|------|------|------|
| 當事人 | （案件專屬） | 被告 | 第一手事實、主觀認知、區分「知道/以為/不知道」三層 |
| 刑事檢察官 | PRO | 偵查/起訴 | 建構最強指控、構成要件分析 |
| 調查局/金流分析師 | INV | 偵查輔助 | 金流拆解、數位證據、技術事實（不做法律論斷） |
| 法官 | JDG | 中立 | 無罪推定、罪疑唯輕、證明力評價、定錨結論 |
| 辯護律師 | DEF | 辯護 | 找邏輯漏洞、定義消除、程序瑕疵 |

**關鍵設計原則：**
- **當事人人格要真實**：注入真實性格（如本案 Jet 的「眼見為憑、不騙人」），使其陳述
  貼近業務現實而非法律修辭。
- **INV 只做技術事實**：金流數字、IP、網域、系統分工——這些是後續「翻盤引擎」的來源。
- **JDG 必須誠實**：不為任何一方背書，這是機率估計可信的根基。

**產出：** `Agent 團隊`（`.claude/agents/` 各角色定義檔）

## 階段 2：特徵點盤點

| 步驟 | 動作 | 要點 |
|------|------|------|
| 2-1 | 通讀當事人全部筆錄 | 找出偵查機關「認定有關聯」的每一個點 |
| 2-2 | 每個特徵建立五欄定義 | 特徵／**認定方式**／**認定定義**／當事人解釋／**模糊空間** |
| 2-3 | 分類分組 | 身份組織／知情／通訊／關鍵工具／與他人關聯／整體歸責 |
| 2-4 | 標記優先級 | 🔴 最高（無法解釋）→ 🟢 較低（有利） |

**核心表格欄位（必備）：**
```
特徵點 | 偵查機關認定方式 | 認定定義/核心主張 | 當事人解釋 | 模糊空間與瑕疵 | 後續研究重點
```

**產出：** `特徵點分析表`（本案 24 點）

## 階段 3：多輪對抗會議

**每輪標準結構：**
```
1. 引入新證據（INV 提供技術事實 / 新調閱的筆錄）
2. PRO 建構/強化指控
3. DEF 提出定義消除或反駁
4. 當事人補充事實視角
5. JDG 逐點定錨：有利 / 中性/待補強 / 不利
6. 盤點本輪「結論變化」：改變/未改變/翻轉
```

**最少三輪原則：** 第一輪開場立場 → 第二輪交叉詰問（引入關鍵數據）→ 第三輪 JDG 收斂。

**結論變化盤點表（必備）：**
```
特徵點 | 各輪結論 | 是否改變 | 變化驅動證據
```

**關鍵心法：**
- **改變的點幾乎都由「技術事實」驅動**（本案：myhomepei 同網域、黑名單 IP 歸屬）。
- **固著於「有利」的點，多屬「主觀切割」與「事後證據」類**——這是最穩防線。
- **固著於「中性」的點，往往是全案勝負手**（本案 F-09 認知盲點）。

**產出：** `各輪會議紀錄` ＋ `結論變化盤點表`

## 階段 4：槓桿點識別與破防（D 點方法）

**什麼是 D 點（決勝/槓桿點）：**
> 單一一個若成立則全案崩、若攻破則全案翻的特徵點。通常是「直接明知證據」或
> 「共謀直接連結」。

| 步驟 | 動作 |
|------|------|
| 4-1 | 從盤點表找出 D 點（最高優先、方向能翻轉全案者） |
| 4-2 | 調閱所有與 D 點相關的文件深化 |
| 4-3 | 建立 D 點的「錨點清單」（支撐 D 點的每個證據 A1, A2…） |
| 4-4 | 逐一檢驗每個錨點的真實性與證明力 |
| 4-5 | 設計「破防」：定義消除 / 證據法則壓制 / 事實校準 |

**破防的三種武器：**
1. **定義消除**：將不利事實重新定義為中性/合法（如「博奕」≠「賭博」法律定性）。
2. **證據法則壓制**：不具結→補強法則；單方記載→不可回推故意。
3. **認知落差**：傳遞者自身認知不正確→受傳遞者無從獲得犯罪明知。

**本案 D1 破防實例（五重削弱）：**
```
D1 = 周正桓稱告知賴宗杰客戶涉博奕、賴答「業界都這樣」
 ① 周正桓拒絕具結 → 補強法則
 ② 博奕≠賭博 → 定義落差
 ③ 周正桓自承法律不懂 → 傳遞者認知不正確
 ④ 「遊戲平台」為產業鏈通用語（史育靜佐證）→ 全鏈盲點
 ⑤ 客觀紅旗不在當事人視野 → 無容任基礎
```

**產出：** `槓桿點深化分析`

## 階段 5：事實校準（防止採信單方有利證據而誤判）

**鐵則（本案第四輪教訓）：**
> 犯罪集團/對造的單方記載（帳冊、表格、內部指示），會以對自己有利或誇大的方式記載
> 與第三人的關係，**不能直接採為該第三人有罪的證據**。每一處提及當事人之處，須以此
> 標準重新檢視。

| 步驟 | 動作 |
|------|------|
| 5-1 | 當事人提供真實事實（推翻某錨點） |
| 5-2 | 區分「身分連結」與「指揮/共謀閉環」——前者常存活，後者常倒塌 |
| 5-3 | 重評受影響的所有特徵點，連鎖回調 |
| 5-4 | 記錄機率軌跡的震盪與校準 |

**產出：** `事實修正後重評`

## 階段 5.5：辨偽普篩（可選，收斂前的總清算）

> 在收斂前插入一次系統性反證過濾。詳見 `辨偽篩選法.md`。

| 步驟 | 動作 |
|------|------|
| 5.5-1 | 織「篩網」：列出檢方所有**不行、錯誤的辨識方法**（以客觀推主觀、單方記載回推故意、未具結唯一證據、定義擴張、身分連結＝支配、經驗法則濫用、提議＝執行…） |
| 5.5-2 | 把 24 特徵點逐點倒進篩子，判定各點依附哪條無效辨識法 |
| 5.5-3 | 對每條無效辨識法跑**反證法三式**（歸謬／卷證矛盾／法則違反） |
| 5.5-4 | 產出「過篩總表」：A 全漏（歸零）／B 降級／C 真殘留（硬點→MIT） |

**產出：** `辨偽過篩總表`（把零散的逐點抽積木，換成一張系統化篩網）

## 階段 6：無罪路徑收斂

| 步驟 | 動作 |
|------|------|
| 6-1 | 將特徵點重分為「因脫鉤弱化」「獨立硬點」「主觀樞紐」「本就有利」 |
| 6-2 | 建立無罪路徑的支柱（結構脫鉤／D 點削弱／責任分層／認知防禦） |
| 6-3 | **依罪名分流估計無罪機率**（不要給單一籠統數字） |
| 6-4 | 區分「實質無罪（免入監之重罪全免）」與「全罪名零定罪」兩種達成定義 |
| 6-5 | 列出待執行補強動作 ＋ 殘餘硬點的切割方案 |
| 6-6 | 收斂成一句話法庭戰略 |

**產出：** `無罪路徑總圖`

---

## 誠實鐵律（貫穿全程）

1. **不為達標而灌水機率**——給會在法庭崩潰的數字，是對當事人失職。
2. **分罪名估計**——重罪與輕罪分開，避免被殘餘硬點拖累整體判斷。
3. **每根支柱都要站得住卷證**——路徑的力量來自可被檢驗，而非樂觀。
4. **硬點正面承擔**——當事人自身行為（如黑名單提議）不可假裝消除，只能切割止損。

```
 ↓

 ↓

 ↓

 ↓
收斂無罪路徑（四支柱）→ 分罪名機率 → 法庭戰略
```
